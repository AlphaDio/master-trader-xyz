Autonomous Base Market Agent

Master Trader XYZ

A Codex-driven agent built to research crypto markets on Base, coordinate with its human operator, and propose guarded execution plans with durable state, audit logs, and structured workflows.

Harness: codex-cli Model: gpt-5.4-mini Network: Base
View Repository Synthesis Skill

Guarded by Design

Execution tasks are wrapped in transaction guardrails, operator approval hooks, and a harness that records structured outputs instead of trusting free-form model text.

Stateful and Auditable

The agent persists run state, inbox requests, skill integrations, secret references, and immutable artifacts so every attempt can be resumed, inspected, and evaluated.

Built for Human Collaboration

When information is missing, Master Trader XYZ blocks cleanly, asks only for the minimum input, and resumes with durable memory instead of restarting from scratch.

What It Does

  • Collects market context and price signals from Base-related sources.
  • Analyzes portfolio state and proposes next actions in structured JSON.
  • Enforces execution guardrails before any live transaction is accepted.
  • Supports dedicated contest tasks such as Synthesis registration without polluting trading tasks.

Current Operating Posture

Dry-run execution mode enabled

The current default posture favors safe analysis, reproducible logs, and explicit human approval over blind automation.

Run the Agent

This project is designed to run locally as a Codex-powered CLI harness. To launch the Synthesis registration workflow, set the workflow config and start a run:

$env:DEFAULT_SKILL_IDS="synthesis" $env:WORKFLOW_CONFIG_JSON=Get-Content .\examples\synthesis-entry-workflow.json -Raw node .\src\index.js run-once
